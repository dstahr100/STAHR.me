    • HTML is Like Writing

     

    The early Internet was designed mainly to share text documents between computers over long distances – that legacy can be seen in HTML today. This is because HTML’s structure is similar to how written documents are structured.

     

    I now know that the process of creating an HTML document is almost the same as developing a well thought out paper. In both situations, you are trying to achieve the same objective – to get a message across.

     

    One of the main ways people communicate with each other is through writing because writing is a great way to share information. This was the case long before computers which can attribute much of their design to writing. Have you ever wondered why computer keyboards are designed the way they are? A computer keyboard is based off of a typewriter. The point here is that HTML is another way to write a written message to someone. So it would make sense that HTML would involve the same principles as writing a paper.

     

    All written documents covey a message through structure by helping people understand what is written. Usually in print, this structure is either a hierarchy of headings and sub-headings or information organized together into different groups. If an article is long and complex, headings allows a reader to more easily make out the message of the article.

     

    Anyone who uses Microsoft Word already knows how to structure documents with headings. Chances are that you have made a heading on Microsoft Word before with its Styles Pane section.

     

    Code Example

     

     The same principles for creating headings in Microsoft Word are used when making a HTML document. The only difference is that instead of the Styles Pane being used to create the headers you use the <h1> , <h2> , and <h3> header HTML tags. Also you use the <p> tags to create paragraphs.

     

    This is what the code looks like for creating headers and paragraphs in a HTML document:

    
<!DOCTYPE html>
<html>
<head>
    <title>Bob 4/1/18</title>
</head>
<body>
    <h1>Heading 1</h1>
    <p>Paragraph Text</p>
    <h2>Heading 2</h2>
    <p>Paragraph Text<p>
    <h3>Heading 3</h3>
    <p>Paragraph Text<p>  
</body>
</html>
 

     

    The code is then processed by a browser to render a webpage as seen in this demo.

     

    See the Pen Demo for Website by David (@Starman5) on CodePen.

     

    Obviously, there are a lot of similarities on how Word Documents and HTML documents are structured. They both use headings to help guide readers, and they both organize information into different groups. For example, the <head> tag used with the <title> tag separates information about the webpage from information that is a part of the webpage. In this case, the information is a name and date which would be displayed in the tab of a browser if it were a full-fledge webpage. Likewise, in a paper, the name and date (or about information) is separated from the main content of the paper. This is because the about information is unrelated to the main content of the paper and is only there to give context about the paper.

     

    I was really amazed from the realization that the skills needed for writing papers and making HTML documents were the same. However, the more I thought about it, the more it made sense. In both situations the main objective that you are trying to do is get a message across to someone. Both require a brainstorming phrase – where you figure out who your audience is and how to best structure your message for that audience. Also, both require a drafting and revision phrase until your message is perfected.

     

    I credit my ability to learn HTML to the fact that I have been taught good writing practices. I was taught that writing a well thought out paper requires structure. Whether it be an introduction paragraph, supporting paragraphs, or a conclusion paragraph, a paper has a lot of structure. The skills that I have learned from writing have allowed me to easily learn HTML. I can now see why good writers can make good coders from my experiences learning HTML. This is because a HTML document is simply another form of writing.
