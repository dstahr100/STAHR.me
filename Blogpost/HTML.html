    • What is HTML?

     

     

    HTML’s main function is to organize webpages by breaking them up into manageable sections called tags. This is a great way to design webpages by being able to mark-up any content any which way into a workable, easy to understand, structure. The acronym HTML even stands for Hyper Text Mark-Up Language.

     

    As a result,  HTML is a mark-up programing language and is the first stop to learning about web development. The concept behind HTML might sound simple at first but the more you build on a website it becomes anything but simple! Look no further than a common newspaper website were for each article and for each different section that article is placed into – like domestic and worlds news –  the website becomes more complex. A typical web user does not realize the order and structure that is need to properly rendered a webpage after a Google search. However there is a lot of structure that is needed and HTML is what provides this structure.

     

    In order to understand how a website is brought to a user – and how HTML plays a role in that process –  we need to first go over the basics. Such as, knowing key web terms and how the World Wide Web works. People access the web through a program called a browser and from many different devices. A browser is software that requests a webpage from a web server and renders it to a user with the help of HTML. Chrome, Safari, and Firefox are all different browsers that can be used to access the web. A web server is a computer, that is always connected to the internet, running specialized software that is designed to fulfill the requests for webpages from browsers.

     

    In addition, a device is needed to access the web. Nowadays many different devices can be used to access the web. These devices can be either be desktop computers, laptops, tablets, smart phones, and the much overlooked screen readers for the disabled. A screen reader is a program that reads the content of a webpage to a user. This allows someone who has a visual impairment or another disability like dyslexia to access a website.

     

    Regardless of the device, and the specific browser that one uses, HTML uses the same structure to describe the content of a webpage to browsers. This common structure of HTML is what helps make sure the content of a webpage is the same no matter how it is viewed. HTML tells a browser how a webpage is structured with tags which are also called elements. The terms tags and elements are used interchangeably.

     

    Code Example

     

     

    A HTML element usually consists of two tags with one opening tag and one closing tag. The resulting HTML element acts as a container that tells browsers information about a webpage by what is contained between its tags.

     

     

    To illustrate, a basic webpage will have the following HTML elements:

    
<!DOCTYPE html>
<html>
    <head>
        <title></title>
    </head>
    <body>
    </body>
</html> 
 

    The <!DOCTYPE html> HTML element tells the browser that HTML5 will be the current HTML version to run in the browser. The <!DOCTYPE html> element is also an example of an HTML element without a closing tag. Most HTML elements though do have closing tags and only specific elements that do not need a closing tag in their use case will not have them. So in the use case of what HTML version should be declared, the <!DOCTYPE html> element only needs a standalone tag for it to accomplish its task.    

     

    The next HTML element is the <html> tag. This type of HTML element does require a closing tag for its use case. The <html> element lets a browser know that HTML code will be found between its opening and closing tags. Therefore, a browser will know that it has to execute HTML code once it sees that tag.

     

    The <head> element contains information about the webpage that is not shown in the main window of a browser. Such information would be what language character set the webpage should be in and other elements like the <title> tag. The <title> tag is the text that is shown on the tab of your browser.

     

    On the other hand, the <body> element contains information that will be shown in the main window of a browser. This can be a whole host of other HTML elements that help render the user interface of a webpage.

     

    The annotated picture below shows how the <head> and <body> tags work on a webpage:

     

     

    The <head> element is annotated in red, while the <body> element is annotated in orange. The <title> tag that is inside the <head> tag is the “Home Page” text which is an example of descriptive information about the webpage that is not displayed in the main window of a browser. Anything else that is between the <body> tags would be information that is displayed in the main window of a browser.

     

    All of these tags that form a basic webpage have a specific purpose. Each creates an orderly structure that browsers then can use to process a webpage’s content. This is the key idea behind HTML and it is the reason why HTML is essential in web development.
